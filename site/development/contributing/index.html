    Guia de Contribuição¶

    Obrigado por considerar contribuir para o Docker Pilot! Este guia fornece todas as informações necessárias para contribuir efetivamente com o projeto.

    Código de Conduta¶

    Este projeto e todos os participantes são governados pelo Código de Conduta. Ao participar, você concorda em seguir este código.

    Comportamento Esperado¶

    • Use linguagem acolhedora e inclusiva
    • Respeite pontos de vista e experiências diferentes
    • Aceite críticas construtivas graciosamente
    • Foque no que é melhor para a comunidade
    • Mostre empatia com outros membros da comunidade

    Comportamento Inaceitável¶

    • Uso de linguagem ou imagens sexualizadas
    • Trolling, comentários insultuosos/depreciativos
    • Assédio público ou privado
    • Publicação de informações privadas sem permissão
    • Qualquer conduta considerada inadequada em ambiente profissional

    Como Contribuir¶

    Existem várias maneiras de contribuir para o Docker Pilot:

    1. Reportar Bugs¶

    Se você encontrou um bug, ajude-nos criando um issue:

    1. Verifique se o bug já foi reportado
    2. Use o template de bug report
    3. Inclua informações detalhadas sobre o ambiente
    4. Forneça passos para reproduzir o problema
    5. Adicione logs e screenshots quando relevante

    Template de Bug Report:

    **Descrição do Bug**
Descrição clara e concisa do problema.

**Passos para Reproduzir**
1. Vá para '...'
2. Clique em '....'
3. Execute '....'
4. Veja o erro

**Comportamento Esperado**
O que você esperava que acontecesse.

**Screenshots**
Se aplicável, adicione screenshots.

**Ambiente:**
- OS: [e.g. Windows, macOS, Linux]
- Docker Version: [e.g. 24.0.7]
- Docker Pilot Version: [e.g. 1.0.0]
- Node.js Version: [e.g. 18.17.0]

**Informações Adicionais**
Qualquer contexto adicional sobre o problema.

    2. Sugerir Melhorias¶

    Para sugerir uma nova funcionalidade:

    1. Verifique se a funcionalidade já foi sugerida
    2. Use o template de feature request
    3. Descreva claramente o problema que resolve
    4. Forneça exemplos de uso
    5. Considere alternativas

    Template de Feature Request:

    **Descrição da Funcionalidade**
Descrição clara e concisa da funcionalidade desejada.

**Problema Resolvido**
Qual problema esta funcionalidade resolve?

**Solução Proposta**
Como você gostaria que funcionasse?

**Alternativas Consideradas**
Quais outras soluções você considerou?

**Contexto Adicional**
Qualquer informação adicional relevante.

    3. Contribuir com Código¶

    Para contribuir com código:

    1. Fork o repositório
    2. Crie uma branch para sua funcionalidade
    3. Faça suas alterações
    4. Adicione testes
    5. Execute os testes
    6. Faça commit das mudanças
    7. Crie um pull request

    Configuração do Ambiente¶

    Pré-requisitos¶

    • Node.js 18.x ou superior
    • npm 9.x ou superior
    • Docker 24.x ou superior
    • Git 2.x ou superior

    Instalação¶

    1. Clone o repositório:
    git clone https://github.com/your-username/docker-pilot.git
cd docker-pilot
    1. Instale as dependências:
    npm install
    1. Configure o ambiente de desenvolvimento:
    # Crie arquivo de configuração local
cp .env.example .env.local

# Instale ferramentas de desenvolvimento
npm run dev:setup
    1. Compile o projeto:
    npm run build
    1. Execute os testes:
    npm test

    Configuração do Docker¶

    # Verificar se Docker está rodando
docker version

# Iniciar Docker (se necessário)

    Padrões de Código¶

    Estilo de Código¶

    Seguimos as convenções do TypeScript e ESLint:

    // ✅ Bom
class ContainerManager {
  private readonly containers: Map<string, Container> = new Map();

  async startContainer(name: string): Promise<void> {
    const container = this.containers.get(name);
    if (!container) {
      throw new Error(`Container ${name} não encontrado`);
    }

    await container.start();
  }
}

// ❌ Ruim
class containerManager {
  private containers: any;

  startContainer(name) {
    return this.containers.get(name).start();
  }
}

    Nomenclatura¶

    • Classes: PascalCase (ContainerManager)
    • Métodos/Funções: camelCase (startContainer)
    • Variáveis: camelCase (containerName)
    • Constantes: UPPER_SNAKE_CASE (DEFAULT_TIMEOUT)
    • Interfaces: PascalCase com prefixo I (IContainerConfig)
    • Types: PascalCase (ContainerStatus)
    • Enums: PascalCase (LogLevel)

    Estrutura de Arquivos¶

    // arquivo: src/utils/ContainerUtils.ts

import { Container, ContainerStatus } from '../types';
import { DockerClient } from '../core/DockerClient';
import { Logger } from './Logger';

/**
 * Utilitários para gerenciar containers Docker
 */
export class ContainerUtils {
  private static readonly logger = new Logger('ContainerUtils');

  /**
   * Verifica se um container está rodando
   * @param container Container a verificar
   * @returns true se estiver rodando
   */
  static isRunning(container: Container): boolean {
    return container.status === ContainerStatus.Running;
  }

  /**
   * Obtém containers por status
   * @param client Cliente Docker
   * @param status Status desejado
   * @returns Lista de containers
   */
  static async getByStatus(
    client: DockerClient,
    status: ContainerStatus
  ): Promise<Container[]> {
    try {
      const containers = await client.listContainers();
      return containers.filter(c => c.status === status);
    } catch (error) {
      this.logger.error('Erro ao obter containers por status', error);
      throw error;
    }
  }
}

    Tratamento de Erros¶

    // ✅ Bom - Tratamento específico
async function stopContainer(name: string): Promise<void> {
  try {
    const container = await findContainer(name);
    if (!container) {
      throw new ContainerNotFoundError(`Container ${name} não encontrado`);
    }

    await container.stop();
    logger.info(`Container ${name} parado com sucesso`);
  } catch (error) {
    if (error instanceof ContainerNotFoundError) {
      logger.warn(`Container ${name} não encontrado`);
    } else {
      logger.error(`Erro ao parar container ${name}`, error);
    }
    throw error;
  }
}

// ❌ Ruim - Tratamento genérico
async function stopContainer(name: string): Promise<void> {
  try {
    const container = await findContainer(name);
    await container.stop();
  } catch (error) {
    console.log('Erro:', error);
  }
}

    Padrões de Teste¶

    Testes Unitários¶

    // arquivo: tests/unit/utils/ContainerUtils.test.ts

import { ContainerUtils } from '../../../src/utils/ContainerUtils';
import { Container, ContainerStatus } from '../../../src/types';
import { createMockContainer } from '../../helpers/mocks';

describe('ContainerUtils', () => {
  describe('isRunning', () => {
    it('deve retornar true para container rodando', () => {
      // Arrange
      const container = createMockContainer({
        status: ContainerStatus.Running
      });

      // Act
      const result = ContainerUtils.isRunning(container);

      // Assert
      expect(result).toBe(true);
    });

    it('deve retornar false para container parado', () => {
      // Arrange
      const container = createMockContainer({
        status: ContainerStatus.Exited
      });

      // Act
      const result = ContainerUtils.isRunning(container);

      // Assert
      expect(result).toBe(false);
    });
  });

  describe('getByStatus', () => {
    it('deve filtrar containers por status', async () => {
      // Arrange
      const mockClient = {
        listContainers: jest.fn().mockResolvedValue([
          createMockContainer({ status: ContainerStatus.Running }),
          createMockContainer({ status: ContainerStatus.Exited }),
          createMockContainer({ status: ContainerStatus.Running })
        ])
      };

      // Act
      const result = await ContainerUtils.getByStatus(
        mockClient as any,
        ContainerStatus.Running
      );

      // Assert
      expect(result).toHaveLength(2);
      expect(result.every(c => c.status === ContainerStatus.Running)).toBe(true);
    });
  });
});

    Testes de Integração¶

    // arquivo: tests/integration/docker/ContainerManager.test.ts

import { ContainerManager } from '../../../src/core/ContainerManager';
import { DockerClient } from '../../../src/core/DockerClient';
import { setupTestEnvironment, teardownTestEnvironment } from '../../helpers/setup';

describe('ContainerManager Integration', () => {
  let containerManager: ContainerManager;
  let dockerClient: DockerClient;

  beforeAll(async () => {
    await setupTestEnvironment();
    dockerClient = new DockerClient();
    containerManager = new ContainerManager(dockerClient);
  });

  afterAll(async () => {
    await teardownTestEnvironment();
  });

  beforeEach(async () => {
    // Limpar containers de teste
    await dockerClient.pruneContainers();
  });

  it('deve criar e iniciar container', async () => {
    // Arrange
    const containerName = 'test-nginx';
    const image = 'nginx:alpine';

    // Act
    await containerManager.create(containerName, image);
    await containerManager.start(containerName);

    // Assert
    const container = await dockerClient.getContainer(containerName);
    expect(container.status).toBe('running');
  });
});

    Testes E2E¶

    // arquivo: tests/e2e/cli/container-commands.test.ts

import { execSync } from 'child_process';
import { setupTestEnvironment } from '../../helpers/setup';

describe('Container Commands E2E', () => {
  beforeAll(async () => {
    await setupTestEnvironment();
  });

  it('deve listar containers via CLI', () => {
    // Act
    const output = execSync('docker-pilot list containers', {
      encoding: 'utf8'
    });

    // Assert
    expect(output).toContain('CONTAINER ID');
    expect(output).toContain('IMAGE');
    expect(output).toContain('STATUS');
  });

  it('deve iniciar container via CLI', () => {
    // Arrange
    execSync('docker run -d --name test-nginx nginx:alpine');

    // Act
    const output = execSync('docker-pilot start test-nginx', {
      encoding: 'utf8'
    });

    // Assert
    expect(output).toContain('Container test-nginx iniciado');
  });
});

    Processo de Review¶

    Checklist do Pull Request¶

    Antes de criar um PR, verifique:

    • Código segue os padrões estabelecidos
    • Testes foram adicionados/atualizados
    • Documentação foi atualizada
    • Builds passam sem erros
    • Não há conflitos de merge
    • Commit messages são descritivos

    Template de Pull Request¶

    ## Descrição

Breve descrição das mudanças realizadas.

## Tipo de Mudança

- [ ] Bug fix (mudança que corrige um problema)
- [ ] Nova funcionalidade (mudança que adiciona funcionalidade)
- [ ] Breaking change (mudança que quebra compatibilidade)
- [ ] Documentação (apenas mudanças na documentação)

## Como Foi Testado

Descreva os testes realizados para verificar as mudanças.

## Checklist

- [ ] Meu código segue os padrões do projeto
- [ ] Realizei self-review do código
- [ ] Comentei código complexo
- [ ] Atualizei documentação
- [ ] Testes passam localmente
- [ ] Adicionei testes que provam que o fix/feature funciona

## Screenshots (se aplicável)

Adicione screenshots para mudanças na UI.

    Processo de Review¶

    1. Automated Checks: CI/CD executa testes e linting
    2. Code Review: Pelo menos um maintainer deve aprovar
    3. Manual Testing: Testes manuais se necessário
    4. Merge: Squash and merge para manter histórico limpo

    strategy:
      matrix:

    - uses: actions/checkout@v3

    - name: Run linter

    - name: Run tests
      run: npm run test:coverage

    - name: Upload coverage
